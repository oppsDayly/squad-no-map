# Security Policy

## Supported Versions

Security fixes are handled for the latest public source version.

## Reporting a Vulnerability

Open a private security advisory on the project host if available. If the repository host does not support private advisories, contact the maintainers through the public issue tracker with a minimal description and ask for a private reporting channel before sharing exploit details.

Do not include private stream captures, access tokens, unpublished model files, or other sensitive local data in public issues.
